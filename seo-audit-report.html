优先级问题

高根路径会按语言请求头或 cookie 改变目标语言和 canonical

影响：/ 同时被 sitemap 标为 x-default 和 en，但请求 Accept-Language: ja 会 307 到 /ja，请求 Accept-Language: es 会 307 到 /es，带 Cookie: NEXT_LOCALE=ja 也会 307 到 /ja。这会让搜索引擎看到不稳定的根 URL，削弱 x-default/default locale 信号，增加重复与错语种索引风险。

证据：src/i18n/routing.ts:15-16 使用 localePrefix: "as-needed" 且 localeDetection: true；src/app/sitemap.ts:17 将 x-default 指向默认语言根 URL。

  • 建议将 localeDetection 改为 false，让 / 永远稳定服务英文默认页，语言切换只通过显式链接和 cookie 用于 UI。
  • 如果需要 x-default 语言选择页，则让 / 成为中立选择页，并把英文改为显式 /en，避免 / 同时承担英文页和语言检测入口。
  • 部署后用无 cookie、不同 Accept-Language、Googlebot UA 分别验证 / 的 status、canonical、hreflang 是否一致。

高多语言静态法务/关于页面内容未本地化，且缺少页面级 metadata

影响：/ja/privacy-policy 的 HTML lang="ja"，但 H1 是英文 “Privacy Policy”，正文也是英文；/es/about 同样是西班牙语页面框架加英文正文。这些 URL 被 sitemap 收录，会造成语言不匹配、低价值本地化页和错误摘要。

证据：src/app/[locale]/privacy-policy/page.tsx:5-8、src/app/[locale]/about/page.tsx:5-7、src/app/[locale]/terms-of-service/page.tsx:5-8 均硬编码英文内容；src/app/sitemap.ts:26 把 /privacy-policy、/terms-of-service、/copyright、/about 放入所有语言 sitemap；src/app/[locale]/layout.tsx:26-39 提供的是站点级默认 metadata，静态页未覆盖 canonical/hreflang。

  • 为这些页面接入 locale messages，至少本地化 H1、正文、title、description。
  • 为每个静态页添加 generateMetadata，输出 page-specific title、description、canonical、languages。
  • 如果暂不翻译法务页，先从非英文 sitemap 移除这些 URL，或对未翻译 locale 加 noindex，避免索引错语种页面。

中首页移动端横向溢出

影响：390px 移动视口下，英文首页 overflowX=true。定位到自动滚动文章卡片的动画轨道，部分卡片被计算到页面可滚动宽度里，导致横向滚动。移动可用性问题会影响体验，也可能影响移动端 SEO 质量信号。

证据：src/app/[locale]/HomePageClient.tsx:103-112 使用 overflow-hidden 容器 + animate-auto-scroll 轨道 + 双倍文章卡片；src/app/globals.css:82-88 通过 translateX(-50%) 动画移动轨道。浏览器检测到首页卡片 right 坐标超过 25000px。

  • 为 carousel 外层增加可靠的横向裁切和布局隔离，例如 overflow-x: clip、max-width: 100%、contain: paint；同时复测 document.documentElement.scrollWidth。
  • 减少实际 DOM 中的重复卡片数量，或改用 CSS/JS 虚拟滚动，不要把全部内容复制两遍。
  • 在 390px、768px、1280px 三个视口加入回归检查：scrollWidth <= clientWidth + 1。

中英文文章 SEO title 普遍过长

影响：所有 40 个英文内容页在拼接站名后标题都超过 60 字符，其中 37 个超过 90 字符。搜索结果中标题大概率被截断，弱化关键词和点击吸引力。样本 /guide/anime-vanguards-beginner-guide title 长度为 96。

证据：src/app/[locale]/[...slug]/page.tsx:75 使用 ${item.metadata.title} - ${localizedSiteName}；英文内容元数据标题本身多为长句。

  • 在 MDX metadata 中新增 seoTitle 字段，H1 保持长标题，title tag 使用更短版本。
  • 避免在已经包含 “Anime Vanguards” 的文章标题后重复追加完整站名；可按页面类型选择性追加品牌。
  • 优先优化高价值页面：codes、tier list、beginner guide、unit guide、calculator。

中首页 HTML 与可见文本量偏大，主意图被稀释

影响：首页渲染约 9071 词，HTML 下载约 437 KB，本地 dev TTFB 粗测约 0.85s。首页同时展示大量文章卡片、模块说明、FAQ、侧边栏和 carousel，容易让核心主题“Anime Vanguards Wiki / codes / units / guide”被大量长尾内容稀释，也增加移动端渲染成本。

证据：src/app/[locale]/page.tsx 收集全部 CONTENT_TYPES 的文章传给首页；src/app/[locale]/HomePageClient.tsx:107-121 又把 articles 复制两遍渲染。

  • 首页 carousel 改为 curated 8-12 个热门页面，不要渲染所有 40 篇内容并复制一份。
  • 把长模块说明压缩为更短的入口卡片，把深度内容留给栏目页。
  • 补充生产环境 Lighthouse / PageSpeed 后再判断是否需要拆分客户端组件或延迟加载视频区块。

中sitemap 的 lastmod 每次使用当前时间

影响：sitemap 中每个 URL 的 lastmod 都来自 new Date()，会让所有 212 个 URL 在每次生成时看起来都刚更新。长期看会降低 lastmod 信号可信度，也可能浪费抓取预算。

证据：src/app/sitemap.ts:38 使用 lastModified: new Date()。

  • 动态文章 URL 使用 MDX lastModified 或 date。
  • 静态页面使用固定更新时间，或只有内容真正变更时更新。
  • 确保 sitemap 输出的是 UTC ISO 时间，且不会因每次请求而变化。

低栏目页 heading 层级跳过 H2

影响：/codes 样本 heading levels 为 1,3,3,3,3,4,4，页面从 H1 直接进入卡片 H3。它不是严重排名问题，但会影响文档结构、辅助技术理解和内容语义。

证据：src/app/[locale]/[...slug]/page.tsx:103 的栏目页卡片标题使用 h3，但前面没有 H2 容器标题。

  • 把栏目页卡片标题改为 h2，或在列表前加一个 h2，再用 h3 作为卡片标题。

低head 中未输出 favicon/manifest 链接

影响：资源文件存在于 public/，但渲染后的 head 没有 rel="icon"、rel="apple-touch-icon" 或 rel="manifest"。这通常不是核心排名因素，但影响浏览器标签、收藏、PWA 和品牌展示。

证据：浏览器 head 检查仅发现 canonical/alternate，没有 icon/manifest；src/app/[locale]/layout.tsx:33-39 metadata 未声明 icons 或 manifest。

  • 在 layout metadata 中添加 manifest: "/manifest.json" 和 icons 配置。
  • 确保 apple touch icon、maskable icon、favicon 尺寸与 public/manifest.json 一致。

robots.txt 允许抓取并指向 sitemap；代表性内容 URL 返回 200；不存在页面样本返回 404。内容目录按 en/es/ja/pt-br 对齐，便于维护 hreflang 对应关系。
